KAHLO ERP — DOCUMENTATION TECHNIQUE EXÉCUTABLE
=============================================

1) INSTALLATION LOCALE
----------------------

Prérequis:
- Docker Engine + Docker Compose v2

Procédure validée (commande cible):
  git clone <repo>
  cd Kahlo-ERP
  cp .env.example .env
  docker compose up --build

Services attendus:
- nginx (point d'entrée)
- frontend (React build statique)
- backend (FastAPI)
- db (PostgreSQL)
- redis
- caldav (Radicale)


2) VARIABLES .ENV
-----------------

Toutes les variables supportées sont listées dans `.env.example` avec valeurs
par défaut recommandées pour le développement.

Groupes:
- Infrastructure: POSTGRES_PASSWORD, REDIS_PASSWORD, HTTP_PORT, HTTPS_PORT
- Sécurité: SECRET_KEY, APP_USERNAME, APP_DEFAULT_PASSWORD,
  ADMIN_FORCE_RESET, SESSION_HOURS, LOGIN_MAX_ATTEMPTS, LOGIN_WINDOW_SECONDS
- CORS/API: CORS_ORIGINS, DATABASE_URL, REDIS_URL
- Intégrations: SUMUP_*, BREVO_*, GEMINI_*, GOOGLE_*, CALDAV_*
- Métier: BOUTIQUE_*, OBJECTIF_CA_MENSUEL, STOCK_*, CRM_*
- Exploitation: BACKUP_*, FACTURES_DIR

Recommandé en production:
- SECRET_KEY forte et unique
- APP_DEFAULT_PASSWORD robuste
- CORS_ORIGINS limité aux domaines front réels
- secrets API non vides seulement si intégration activée


3) ADMIN
--------

Création admin initial:
- Au startup backend, si table utilisateurs vide:
  - username = APP_USERNAME
  - mot de passe = APP_DEFAULT_PASSWORD (ou APP_PASSWORD_HASH)

Reset mot de passe admin:
- ADMIN_FORCE_RESET=true
- redémarrer backend
- remettre ADMIN_FORCE_RESET=false


4) DATABASE
-----------

Migrations:
- Exécution automatique d'Alembic au démarrage backend: `upgrade head`
- Fallback create_all en cas d'indisponibilité Alembic

Seed:
- Insertion fournisseurs de base si table vide
- Bootstrap admin si table utilisateurs vide

Sauvegarde:
  docker compose exec db pg_dump -U kahlo kahlo > backup.sql

Restauration:
  cat backup.sql | docker compose exec -T db psql -U kahlo kahlo


5) TESTS
--------

Backend:
  .venv/bin/python -m pytest backend/tests -q

Frontend:
  cd frontend
  npm run build


6) PRODUCTION
-------------

Reverse proxy:
- Service nginx exposé via HTTP_PORT/HTTPS_PORT.
- Ajuster certificats dans `nginx/ssl`.

CORS:
- Paramètre backend `CORS_ORIGINS` (liste CSV).

Variables à adapter a minima:
- POSTGRES_PASSWORD, REDIS_PASSWORD, SECRET_KEY,
  APP_DEFAULT_PASSWORD, CALDAV_PASSWORD, CORS_ORIGINS.
- BIND_HOST=127.0.0.1 si la stack est derrière un reverse proxy externe.


7) EXPLOITATION
---------------

Logs:
  docker compose logs -f backend
  docker compose logs -f nginx
  docker compose logs -f db

Rotation secrets:
- Changer dans `.env`
- redéployer `docker compose up -d --build`
- invalider les sessions (changement SECRET_KEY)



Attention volumes:
- `docker compose down -v` supprime les données persistées (PostgreSQL, Redis, uploads, factures, CalDAV, backups).
